# MAX Polubot Deployment

## Required Hugging Face secrets

Set these in Space Settings -> Secrets:

- `SESSION_JSON` - generated by `python auth.py`, must be WEB/QR session.
- `ADMIN_IDS` - comma-separated MAX user ids allowed to run commands. If you do not know it yet, start with it empty and use `!ктоя`, or use `python list_chats.py` with a valid session.
- `DATABASE_URL` or `NEON_DATABASE_URL` - Neon PostgreSQL connection string.

Optional AI provider secrets:

- Gemini: `AI_PROVIDER=gemini`, `GEMINI_API_KEY`, `GEMINI_MODEL=gemini-3.1-pro`.
- OpenAI-compatible: `AI_PROVIDER=openai`, `OPENAI_API_KEY`, `OPENAI_MODEL=gpt-5.5`, `OPENAI_BASE_URL=https://api.byesu.com`, `OPENAI_WIRE_API=responses`.
- DeepSeek: `AI_PROVIDER=deepseek`, `DEEPSEEK_API_KEY`, `DEEPSEEK_MODEL=deepseek-v4-pro`, optional `DEEPSEEK_BASE_URL`.

Default OpenAI-compatible settings:

- `OPENAI_REASONING_EFFORT=medium`
- `DISABLE_RESPONSE_STORAGE=false`

Hugging Face usually provides `PORT`; the app uses `WEB_PORT` first, then `PORT`, then `7860`.

## Neon notes

Use the pooled or direct Neon connection string as `DATABASE_URL` / `NEON_DATABASE_URL`.
The app adds `sslmode=require` and `connect_timeout` when they are missing.

Default pool settings are serverless-friendly:

- `DB_POOL_MIN_SIZE=0`
- `DB_POOL_MAX_SIZE=5`

## Chat setup commands

Only users from `ADMIN_IDS` can run most commands. When `ADMIN_IDS` is empty, only bootstrap id commands `!ктоя` and `!чат` are accepted.

- `!настройка` - show current runtime settings with secrets masked.
- `!настройка target_chat_id here` - set the current chat as report/reply target.
- `!настройка ai_provider openai`
- `!настройка openai_api_key sk-...`
- `!настройка openai_model gpt-5.5`
- `!настройка openai_base_url https://api.byesu.com`
- `!настройка openai_wire_api responses`
- `!настройка openai_reasoning_effort medium`
- `!настройка disable_response_storage false`
- `!настройка ai_provider deepseek`
- `!настройка deepseek_api_key ...`
- `!настройка deepseek_model deepseek-v4-pro`
- `!настройка ai_provider gemini`
- `!настройка gemini_api_key ...`
- `!проверка` - active DB/MAX/AI checks.
- `!статус` - quick runtime status.

Settings changed through chat are stored in Neon `app_settings` and survive restarts.

`DATABASE_URL`, `NEON_DATABASE_URL`, `SESSION_JSON`, and `ADMIN_IDS` are intentionally env-only bootstrap settings.

## Getting chat and user ids safely

Preferred safe option after `SESSION_JSON` is ready:

```powershell
python list_chats.py
```

This script is read-only: it uses the existing web session, does not run auth, does not touch `.max_session`, does not call logout, and does not send messages to chats. It prints `CHAT_ID` values and any `USER_ID` values found in the login payload.

Inside MAX, the safe in-chat option is:

- `!me` or `!ктоя` - shows your sender id from the already received message event.

- `!чат` - shows the current chat id from the already received message event.
